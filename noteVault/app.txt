Database: 
 - contains 2 entities: users and notes

 users: 
  - id
  - name
  - email (unique)
  - bcrypt hashed password
  - creation date and time

 notes:
  - id
  - user_id
  - title
  - content
  - creation date and time

Frontend:
 - static pages: home
 - user login page, register/profile update
 - profile view page with delete button
 - notes list view page
 - noted edit/update page
 - page for read ony for reading mode
 
 optional: 
  - reset password
  - ai enhance feature

Backend:
 - Connect with database and 2 models, users and notes
 - auth api: 
  - get: for profile view page, user info
  - login and register via post, put(update profile), delete
 - notes api:
  - get: / for list of all user notes and post / for new notes
  - get, put and delete operation on '/:id'
 - take care of authentication and authorization via middleware:
  - create and verify jwt tokens, send token as httpOnly cookie
  - use the token for user id on every request except login and register
